# AAA1000 A+B Problem

[linking](https://www.smqyoj.com/p/AAA1000)

**时间限制：** 1.0 秒

**空间限制：** 512 MB

## 题目描述

这是一道简单的 A+B 题目，大概吧。

输入两个整数，输出它们的和。

## 输入格式

从标准输入读入数据。

一行，包括两个整数，用一个空格间隔。

## 输出格式

输出到标准输出。

一行，包括一个整数。

```input1
1 2
```

```output1
3
```

## 数据范围

对于 $40\%$ 的数据，保证输入的两个整数均在 `int` 范围内。

对于 $100\%$ 的数据，保证输入的两个整数均在 `long long` 范围内。

## 提示

对于使用 `C++` 的用户，猜猜直接提交以下代码的话，能得多少分？

```c++
#include <iostream>
int main()
{
    int a, b;
    std::cin >> a >> b;
    std::cout << a + b << std::endl;
    return 0;
}
```

`int` 变量范围：$-2^{31}\le x \le 2^{31}-1$，其中 $2^{31}=2147483648$。

`long long` 变量范围：$-2^{63}\le x\le 2^{63}-1$，其中 $2^{63}=9223372036854775808$。

如果你获得了 40 分，请注意输入整数的范围。

如果你获得了 50 到 80 分，请注意输出整数的范围。

如果你获得了 90 分，再想想有没有其他边界情况？

此外，还可以尝试一些旁门左道，但是只使用通用的基础数据类型完全可以通过本题。

再试试看用 `Python`，会得到完全不一样的结果，为什么会这样呢？

## Solution

### Hack

- 用 `double` 存 `long long` 的和，会精度丢失，`long long` 是 64 位整数，`double` 只有 53 位有效二进制位，表示不了所有 64 位整数。一旦测试数据接近 long long 的边界，转成 `double` 再相加，结果出错。
- 两个 `long long` 相加本身也可能溢出

### Key

- `__int128` 是 GCC/Clang 提供的一个扩展类型（不是标准 C++ 的一部分），用来表示 128 位整数，能表示范围大约在 $-2^{127}$ 到 $2^{127}-1$ 的整数，完全能容纳两个 `long long` 的和，不会溢出。
- 它没有现成的输入输出支持，必须自己写函数来把它转成字符串输出（比如不断 %10 取余，拼接字符）。
- 在大多数 OJ 环境里，GCC/Clang 都支持 `__int128`，所以用它来解决 A+B problem 的溢出问题是最简洁的办法。

```c++
#include <bits/stdc++.h>
using namespace std;

void print_int128(__int128 x)
{
    if (x == 0) {cout << 0; return;}
    if (x < 0) {cout << '-'; x = -x;}
    string s = "";
    while (x > 0)
    {
        s += '0' + (x % 10);
        x /= 10;
    }
    reverse(s.begin(), s.end());
    cout << s;
}

int main()
{
    long long a, b;
    cin >> a >> b;
    __int128 sum = (__int128)a + (__int128)b;
    print_int128(sum);
    return 0;
}
```
