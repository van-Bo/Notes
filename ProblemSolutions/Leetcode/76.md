# Leetcode 76. Minimum Window Substring

[Problem Link](https://leetcode.cn/problems/minimum-window-substring/description/)

## Hint

`C++ std::unordered_map` 中 `[]` 运算符的底层机制上存在一个重要的副作用：当使用 `[]` 运算符访问一个不存在的键时，会自动将该键插入到 map 中，并为其赋予一个默认值（对于 `int` 类型，默认值是 0）。这意味着，如果在代码中不小心访问了一个不存在的键，可能会导致 map 的大小增加，并且可能会引入意外的行为。

在 `C++` 的 `map` 或 `unordered_map` 中，如果只是想检查一个键是否存在，绝对不要使用 if (map[key]) 这种写法。正确的检查方法有两种：
    1. 使用 `count()`： `if (map.count(key))` （推荐，代码最简洁）
    2. 使用 `find()`： `if (map.find(key) != map.end())` （性能略微好一点点，因为有些场景可以用迭代器直接操作后续步骤）

## solution

```C++
#include <iostream>
#include <unordered_map>
using namespace std;

string minWindow(string s, string t) {
    using namespace std;
    unordered_map<char,int> need;
    unordered_map<char,int> window;
    int left = 0, right = 0;
    int start = 0, len = INT_MAX;
    int valid = 0;  // 记录窗口中满足 need 条件的字符个数

    for(auto c: t) need[c] ++;

    while(right < s.size())
    {
        char c = s[right ++];
        if(need.count(c))   // 需要字符 c
        {
            window[c]++;
            if(window[c] == need[c]) valid ++;
        }
        
        while (valid == need.size())    // 当前窗口满足条件，尝试收缩左边界，以至于不满足条件
        {
            if (right - left < len) // 更新维护最小区间长度 len，先前 start 已经自增，这里 len 无需加 1
            {    
                len = right - left;
                start = left;
            }
            // 收缩左边界
            char d = s[left ++];
            if(need.count(d))   // 需要字符 d
            {  
                if (window[d] == need[d]) valid --; // 破坏了窗口的有效性
                window[d] --;
            }
        }
    }
    return len== INT_MAX ? "" : s.substr(start, len);
}

int main()
{
    string s = "ADOBECODEBANC";
    string t = "ABC";
    cout << minWindow(s, t) << endl;

    unordered_map<char, int> m;
    m['A'] = 1;
    cout << m['B'] << endl;
    system("pause");
    return 0;
}
```
